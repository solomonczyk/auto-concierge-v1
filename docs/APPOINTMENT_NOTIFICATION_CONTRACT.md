# Appointment Notification Contract

Единый контракт уведомлений по записям: кто получает, на каком событии, и что не дублируется.

---

## 1. События, которые шлют уведомления

| Событие | Описание |
|---------|----------|
| `appointment_created` | Новая запись создана (клиент забронировал) |
| `appointment_confirmed` | Запись подтверждена (менеджер/клиент) |
| `appointment_cancelled` | Запись отменена |
| `appointment_no_show` | Клиент не явился (авто или вручную) |
| `appointment_waitlist_promoted` | **Не выделен** на данном этапе. Переход `waitlist` → `new` обрабатывается как обычный status change. Отдельное событие — при необходимости позже. |

---

## 2. Получатели по каждому событию

| Событие | Клиент | Менеджер | Примечание |
|---------|--------|----------|------------|
| `appointment_created` | да | да | Клиент — подтверждение брони. Менеджер — алерт о новой записи. Не шлём менеджеру при waitlist (только при обычном бронировании со слотом). |
| `appointment_confirmed` | да | нет | Клиент узнаёт, что запись подтверждена. |
| `appointment_cancelled` | да | да | Оба должны знать об отмене. |
| `appointment_no_show` | нет | да | Клиент не явился — уведомляем менеджера. Клиенту не шлём. |
| `appointment_waitlist_promoted` | — | — | Не реализовано как отдельное событие. |

---

## 3. Каналы

| Канал | Статус | Описание |
|-------|--------|----------|
| **Telegram** | active | Основной канал. `NotificationService` → `bot.send_message`. |
| **in-app** | active | Real-time через WebSocket (`appointment_status_changed`, `appointment_created`). Dashboard обновляется без отдельного push. |
| **email** | planned | Не реализован. |

---

## 4. Правила идемпотентности

- **Одно и то же событие не шлёт дубль.** Один state transition = максимум одно уведомление по каждому каналу/получателю.
- **Повторный PATCH в тот же статус не создаёт новое уведомление.** Если `old_status === new_status`, уведомление не отправляется (state machine уже отклоняет такой переход).
- **Уведомление отправляется только после успешного state transition.** Сначала `db.commit()`, потом `asyncio.create_task(notify_...)`. При ошибке перехода уведомление не шлётся.
