# Change Decision Matrix

Правило выбора слоя для новых изменений.

| Тип изменения | Куда вносить | Пример |
|---|---|---|
| Полезно большинству клиентов | Core | Новый универсальный endpoint |
| Меняет поведение без новой логики | Configs | Feature flag, branding, preset |
| Нужна интеграция с внешней системой клиента | Extensions | CRM, телефония, webhook adapter |
| Нужно одному клиенту | Extensions | Кастомный workflow |
| Улучшает безопасность/масштабируемость платформы | Core | RBAC, rate limiting, audit |
| Секреты и приватные ключи | Не в repo | `.env`, secret manager |

## Быстрое правило

Спроси:

1. Это нужно большинству клиентов?
2. Это меняет поведение без переписывания логики?
3. Это связано с конкретным клиентом или внешней системой?

Если:
- да на (1) → Core
- да на (2) → Configs
- да на (3) → Extensions
