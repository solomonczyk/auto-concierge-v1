# Customization Rules

Правила кастомизации Auto-Concierge Core.

Цель: сохранить чистоту core и избежать расхождения версий продукта.

---

## 1. Что можно менять через Configs

Без изменения кода:

- branding / theme
- feature flags
- tenant presets
- integration settings templates
- env configuration

---

## 2. Что реализуется через Extensions

Если требуется новая логика для конкретного клиента:

- CRM интеграции
- телефония
- кастомные workflow
- дополнительные webhook обработчики
- нестандартные отчёты
- внешние API

Такие изменения должны находиться в каталоге:

extensions/

---

## 3. Что запрещено менять напрямую в Core

Следующие компоненты считаются частью платформы и не должны модифицироваться под клиента:

- auth
- tenants
- users / roles
- clients
- appointments
- websocket system
- webhook core
- billing / limits
- observability

Если требуется изменение этих компонентов — оно должно быть универсальным для всех клиентов.

---

## 4. Когда изменение идёт в Core

Фича добавляется в core, если:

- она полезна для большинства клиентов
- она улучшает платформу
- она не ломает текущие API

---

## 5. Когда изменение идёт в Extension

Фича остаётся extension, если:

- она нужна одному клиенту
- она интегрирует внешнюю систему
- она меняет бизнес-процесс конкретного клиента
