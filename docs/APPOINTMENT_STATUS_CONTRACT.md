# Appointment Status Contract

Единый контракт статусов записи для backend, frontend, Kanban, уведомлений и документации.

---

## 1. Финальный список статусов

| Статус | Описание |
|--------|----------|
| `new` | Новая запись, ожидает подтверждения |
| `confirmed` | Подтверждена менеджером/клиентом |
| `in_progress` | В работе (клиент на месте) |
| `completed` | Выполнена |
| `cancelled` | Отменена |
| `no_show` | Клиент не явился |
| `waitlist` | Лист ожидания (нет свободного слота) |

---

## 2. Terminal (конечные) статусы

После перехода в эти статусы запись не возвращается в активный поток:

- `completed`
- `cancelled`
- `no_show`

---

## 3. Kanban — колонки

**Основные колонки доски (слева направо):**

| id | Название |
|----|----------|
| `waitlist` | Лист ожидания |
| `new` | Новая |
| `confirmed` | Подтверждена |
| `in_progress` | В работе |
| `completed` | Готова |

**Отдельно: waitlist**

`waitlist` — это **колонка Kanban** (первая слева). Записи в листе ожидания отображаются на основной доске и могут быть переведены в `new`, когда появится слот.

**Terminal статусы на доске**

`cancelled` и `no_show` — **не отображаются** в основных колонках. Могут быть вынесены в отдельную секцию «Архив» / «Отменённые» или скрыты (зависит от UI).

---

## 4. Единое правило нейминга

- **Backend** использует `completed`, **не** `done`.
- Во всех API, UI, docs, WS-событиях — только `completed`.
- Алиас `done` **запрещён** в новом коде.

---

## 5. Переход в waitlist

**Кто переводит:** клиент (WebApp/бот) при бронировании без выбора слота.

**Условия:**
- Клиент выбирает услугу и дату, но **не выбирает время** (или слоты заняты).
- Флаг `is_waitlist: true` в payload → статус `waitlist`.
- Менеджер/админ позже переводит `waitlist` → `new`, когда появляется слот.

---

## Known mismatch

Места, где ещё встречается `done` или устаревшие значения — требуют правки:

| Файл | Проблема |
|------|----------|
| `frontend/src/hooks/useAppointments.ts` | Union type: есть `"done"` (удалить), нет `"no_show"` (добавить) |
| `Database Schema.md` | Упоминает `done` — заменить на `completed` |
| `backend/alembic/versions/b5bc8092c6f4_add_user_model.py` | Историческая миграция с enum `DONE` — не трогать (уже заменено последующими миграциями) |
